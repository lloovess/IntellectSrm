### 🏗️ Архитектура и планирование

#### [PRD.md](./PRD.md) - **ОБЯЗАТЕЛЬНО ЗАПОЛНИТЬ**
- **Что**: Product Requirements Document - требования вашего продукта
- **Зачем**: Это основной источник истины для ИИ агентов
- **Содержит**:
  - Видение и цели продукта
  - Целевая аудитория
  - Ключевые функции (MVP)
  - User stories
  - Метрики успеха
  - Технический стек
  - План по фазам
- **Первый шаг**: Заполните этот файл перед началом разработки
- **Время на заполнение**: 30-60 минут

#### [CLAUDE.md](./CLAUDE.md) - **ОБНОВИТЬ ДЛЯ ВАШЕГО ПРОЕКТА**
- **Что**: Guidance для ИИ агентов о вашей архитектуре
- **Зачем**: Помогает ИИ правильно находить информацию и следовать паттернам
- **Содержит**:
  - Высокоуровневая архитектура
  - Структура проекта
  - Основные компоненты
  - Команды разработки
  - Ключевые паттерны
  - Где найти что
  - Best practices
- **Обновить**: Замените placeholder значения на специфику вашего проекта
- **Время на обновление**: 20 минут

#### [ARCHITECTURE.md](./ARCHITECTURE.md) - **ДЛЯ УГЛУБЛЕННОГО ПОНИМАНИЯ**
- **Что**: Детальное описание архитектуры системы
- **Зачем**: Помогает разработчикам и ИИ агентам понять "как всё работает"
- **Содержит**:
  - System overview диаграммы
  - Архитектурные паттерны (MVC, DI, Repository)
  - Компонент breakdown
  - Data flow диаграммы
  - Database schema
  - Caching strategy
  - Authentication & Authorization
  - Error handling
  - Scalability considerations
  - Monitoring & Observability
- **Когда читать**: Когда хотите глубоко понять архитектуру
- **Время на чтение**: 1-2 часа
